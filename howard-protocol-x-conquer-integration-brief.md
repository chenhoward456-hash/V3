# Howard Protocol × CONQUER 配合架構技術簡報

## 背景

Howard Protocol（你的系統）和 CONQUER（Charlie / 的系統）要做配合，不是整合。兩邊各自獨立運作，透過標準化的週報告作為資料交換介面。

---

## 兩個系統的定位

### Howard Protocol — 數據收集 + 分析引擎（C2B）
- 學員每日回報：飲食（巨量營養素）、體重、心情、睡眠、精力、補品服從、飲水
- 後台自動統整：趨勢圖、偏差分析、血檢追蹤、荷爾蒙警報、恢復指標
- 產出：週報告、月報告、教練備註
- 學員入口：LINE 官方帳號（學員在 LINE 上回報，系統後端接收）
- 技術棧：Next.js 14 + Supabase + Vercel

### CONQUER — 計畫設計 + 執行方案（B2C）
- 教練端工具：根據數據 + 體態照片 → 產出訓練課表、營養計畫
- 備賽階段管理：消耗期、低碳期、碳載期、高峰期，每個階段有不同巨量設定
- 每日計畫細節：餐次、食物內容、碳水/蛋白質/脂肪/熱量、水分與鈉控制、補品時程
- 功能：匯出 PDF 報告、學員照片收集、訓練課表生成
- Charlie 和 Miles 會用自己的經驗微調 AI 產出的計畫

---

## 資料流

```
學員每日回報（LINE 官方帳號）
        ↓
Howard Protocol 後台自動統整
        ↓
產出「訓練端週摘要」（標準化格式）
        ↓
丟給 CONQUER / Charlie 團隊
        ↓
Charlie 結合體態照片 + 週摘要數據
        ↓
產出下週訓練課表 + 營養計畫（CONQUER）
        ↓
透過 LINE 官方帳號發回給學員
        ↓
學員看到進度 + 新計畫 → 繼續每日回報（循環）
```

## 需要開發的功能

### 1. 訓練端週摘要（新的報告格式）

目前後台已有「📋 近 14 天報告」+ 複製按鈕，但那是教練自己看的完整版。需要新增一個「訓練端摘要」版本，只包含 CONQUER 那邊做決策需要的資訊：

```
【訓練端週摘要】2026/3/28 - 4/3
學員：陳胤豪 ｜ 減脂期 ｜ 距比賽 114 天

■ 體重：83.7 kg（週 -0.8 kg / -1.0%）
■ 體脂：8.4-9.6%（最近量測 3/22）

■ 營養達標率：
  蛋白質：25%（平均 127g / 目標 177g，偏差 -28%）⚠️
  熱量：-7%（2189 / 2347 kcal）
  碳水：+3%（268 / 259g）
  脂肪：+7%（72 / 67g）

■ 恢復指標：
  睡眠 4.8 / 精力 4.8 / 心情 4.8
  推日隔天精力最低（4.0）⚠️

■ 訓練概況：
  14天訓練 8 天（推2、拉2、腿2、肩1、有氧1）
  平均 RPE 7.3｜多次 RPE ≥ 9 ⚠️

■ 荷爾蒙警報：
  睪固酮 404 ng/dL（↓35%）
  游離睪固酮 72.8 pg/mL（↓41%）
  Bioavailable T 182 ng/dL（↓44%）
  → 建議考慮降訓練量或安排 Deload

■ 教練備註：
  蛋白質嚴重不足，需優先解決
  體重終於開始下降，維持當前熱量赤字
```

**實作方式**：在 overview 頁面加一個按鈕「複製訓練端摘要」，點擊後產出上述格式的純文字，可直接貼到 LINE 群組。

### 2. LINE 官方帳號整合

Charlie 提到可以用 LINE 官方帳號讓學員回報，這是學員端的入口。需要：

- LINE Messaging API webhook → 接收學員訊息
- 學員回報格式：可以是自然語言（「今天吃了 2000 大卡，蛋白質 150g」）或結構化表單（LINE Flex Message）
- 後端解析 → 寫入 Supabase 對應表
- 自動提醒：如果學員超過 2 天沒回報，LINE 自動推播提醒
- 週報推播：每週日自動從後台 pull 數據 → 生成摘要 → LINE 推播給學員

### 3. 自動週/月回饋

Howard 在對話中提到「跟自動產稱週回饋跟月回饋」：

- **週回饋**（給學員看的版本）：你這週的表現摘要 + 進步/退步 + 下週建議重點
- **月回饋**：30 天趨勢總結 + 體重/體脂變化 + 血檢追蹤 + 里程碑

這可以用已有的「生成教練建議」AI 功能延伸，加一個學員面向的版本。

### 4. 升級提醒機制

對話中提到「如果有升級還可以一直提醒要記錄」：

- Free tier 學員如果連續使用 7 天，推播升級提示
- 記錄中斷時，提醒語氣根據 tier 不同：Free 溫和提醒 + 升級暗示，499/2999 直接提醒

Charlie 的 CONQUER 介面（ANNA 賽前計畫範例）：

  - 消耗期：碳 58g / 蛋 139g / 脂 46g

- 可匯出 PDF 報告
- 比賽日特殊飲食：多餐制（優格+米糊+雞蛋白+藍莓等）

## Charlie 的關鍵評價

> 「你那個就是一個基準值，沒頭緒、沒方向就照上面就好」

意思是：Howard Protocol 的 AI 報告和建議是起點/baseline，Charlie 團隊會用自己的備賽經驗去微調。AI 的備賽邏輯他「不太敢全信」，但作為數據基礎很有價值。

這個定位很健康：**你的系統負責 what is（數據現況），CONQUER 負責 what to do（計畫調整）**。
